# PRCGSX1TurboZ
PRCGSフォーマットのX1TurboZ用ローダー

## 制限
320x200までのデータしか表示出来ません。  
Cのみで書いていることもあって動作はかなり遅いです。  
最低限の最適化はしていますが･･

## ビルド用環境
Windows上でz88dkを想定しています。  
(Makefileはnmake用です)  
独自命令は画面クリアくらいしか使っていないので、他環境でのビルドもそこまで難しくないと思います。

## 想定使用環境
LSX-Dodgers  
ファイルの読み書き部分とデータのデコード、表示は完全に分離しているのでS-OS等の環境でも動くようにすることができるかもしれません。

## PRCGSとは
こちらに詳細が出ていますが、90年代のアマチュア無線を利用したパケット通信網で提唱された画像フォーマットです。  
https://maaberu.web.fc2.com/prcgs.htm

### PRCGSフォーマットについて覚え書き
上記サイトに書かれていますが、ランレングスデータ周りの記述がわかりにくいので書いておきます。  
ヘッダ内に7つのランレングスデータが用意されていますが、これはそれぞれピクセルデータ内の1-7に対応して、0は長さ1で固定です。  
（ヘッダ内のデータが0-6に対応して7が長さ1ではない)  
つまり、0x2fというピクセルデータがあった場合、ランレングステーブル番号は1ですが見るのはヘッダの最初のランレングスデータです。
